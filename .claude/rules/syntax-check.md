# 代码修改后自动语法检查

每次使用 `Edit` 或 `Write` 修改/创建 `.ts` / `.tsx` / `.vue` / `.js` / `.mjs` / `.cjs` 文件后，**必须**做一次语法检查确认无编译错误。

## 规则

1. 修改/创建任意 `.ts` / `.tsx` / `.vue` / `.js` / `.mjs` / `.cjs` 文件后，运行 `node --check` 或对应工具对被修改文件做语法检查；前端 `.vue` 文件优先用 vue-tsc 链路。
2. 若检查返回错误，**立即修复**，然后再次运行确认错误已消除。
3. 只有确认零错误后，才可以向用户报告操作完成。
4. 若同一次操作修改了多个文件，对所有被修改文件统一检查一次即可。

## 不需要检查的情况

- 只读操作（`Read`、`Grep` 等）
- 修改的是 `.md`、`.json`、`.yaml`、`.txt` 等非代码文件
