## 修复保存通用设置时覆盖项目配置的问题

**问题**：在设置弹窗中修改主题或语言后，`.git-commit-tool.json` 文件中的项目配置被重置为默认值。

**原因**：`save-general-settings` API 调用 `configManager.loadConfig()` 获取配置，但 `loadConfig()` 返回的是合并后的配置（不包含项目级别的原始配置）。当调用 `saveConfig()` 时，项目配置会被 `defaultConfig` 覆盖。

**改动文件**：
1. `src/config.js`
   - 导出 `readRawConfigFile` 和 `writeRawConfigFile` 函数

2. `src/ui/server/routes/config.js`
   - 修改 `POST /api/config/save-general-settings` 端点
   - 使用 `readRawConfigFile()` 直接读取原始配置，避免项目配置丢失
   - 使用 `writeRawConfigFile()` 直接写入，保留所有现有配置

## 修复主题切换与设置弹窗状态不同步的问题

**问题**：设置弹窗中显示的主题与实际界面主题不一致，导致保存设置时不调接口。

**原因**：
1. `ThemeSwitcher.vue` 使用 `localStorage.setItem('theme', ...)` 保存主题
2. `configStore` 使用 `localStorage.getItem('zen-gitsync-theme')` 读取主题
3. 两者使用不同的 localStorage key，导致状态不同步

**改动文件**：
1. `src/ui/client/src/components/ThemeSwitcher.vue`
   - 改用 `configStore` 管理主题状态
   - 添加 `watch` 监听 `configStore.theme` 变化
   - 切换主题时更新 `configStore.theme`，自动同步到 localStorage

2. `src/ui/client/src/components/GitGlobalSettingsDialog.vue`
   - 修复保存逻辑，使用 `!== undefined` 判断是否有设置项需要保存
