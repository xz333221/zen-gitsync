## Pull 失败错误弹窗

**需求**：当 git pull 失败时，弹出对话框展示完整错误信息（之前只有 ElMessage toast，文字截断）。

**改动文件**：
1. `src/ui/client/src/stores/gitStore.ts` - `gitPull()` 返回值从 `boolean` 改为结构化对象 `{ success, needsMerge, error, fullError, pullOutput }`，成功仍弹 ElMessage，失败不再弹 toast 由调用方处理
2. `src/ui/client/src/views/components/GitStatus.vue` - `handleGitPull` 检测失败时弹 `CommonDialog` 展示详细错误；添加 `pullErrorDialogVisible` / `pullErrorInfo` 响应式状态及弹窗模板和样式
3. `src/ui/client/src/components/buttons/GitOperationsButton.vue` - 同上，导入 `InfoFilled`、`CommonDialog`，增加弹窗状态和模板样式
4. `src/ui/client/src/lang/zh/index.js` 和 `en/index.js` - 补充 `@13D1C:` 和 `@76872:` 命名空间下的 pull 错误弹窗国际化条目（需要合并更改/拉取失败/错误详情/我知道了 等）
4. `src/ui/client/src/components/PushProgressModal.vue` - 适配 `gitPull()` 返回值变化（`result.success` 替代 `success`）

**关键结论**：`needsMerge:true` 时（本地有改动导致 pull 失败）弹黄色警告样式弹窗；其他错误弹红色错误样式弹窗；错误文本用 `<pre>` 标签滚动展示。
